

>> 'Role' 모델을 사용하지 않는 이유
Attribute에 룰을 선언해서 해서 계정의 권한을 체크하는 방법이 일반적이긴 하지만
이 방법은 권한간에 높낮이만 있을때만 유요한 방법이다.

만약 권한간에 기능 차이만 있는 경우(예> 게시판1의 관리권한과 게시판2의 읽기 권한이 있는 경우)
는 이 방법으로 처리를 하려면 복잡해진다.

또한 권한이 있는 사람과 없는 사람이 모두 접근이 가능한경우 2중으로 권한처리를 해야 하는 문제가 발생한다.

그래서 별도의 속성을 만들지 않고 처리한다.
(나중에 공통 처리를 위해 미들웨어로 만들 가능성은 있음)
